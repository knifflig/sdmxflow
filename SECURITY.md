# Security Policy

## Supported versions

This project is currently in early development (0.x). Security fixes, if needed, will be released as patch versions.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for potential security vulnerabilities.

Instead, use one of the following:

1. GitHub Security Advisories (preferred): use "Report a vulnerability" if enabled for the repository.
2. Otherwise, contact the repository owner via GitHub (see Contact in the README) and include:
   - a description of the issue
   - impact assessment (what an attacker could do)
   - steps to reproduce
   - any proposed fix or mitigation (if you have one)

We will acknowledge receipt and work with you on a coordinated disclosure.
